## 技术实现：

### 项目简介

本项目是一个基于 Web 端的可交互式个人足迹地图。旨在通过可视化的方式，记录并展示个人去过的城市或国家。项目采用现代化的“前后端分离 + Serverless”架构，确保足迹数据在多台设备间实时同步且永久保存，同时作为了解现代 Web 开发、API 交互以及 Git/GitHub 部署流程的实战演练。

### 技术架构

本项目采用纯前端托管结合云端数据库的轻量级架构：

- **前端展示与交互 (Frontend):**
  - **核心语言**: HTML / CSS / JavaScript
  - **地图渲染**: 计划采用 ECharts 或同等成熟的可视化组件库进行地图绘制。
  - **功能职责**: 负责渲染地图界面、响应用户的点击交互，并提供“添加/删除”足迹的 UI 控件。
- **数据存储与接口 (Backend/Database):**
  - **核心服务**: Supabase (或同类免费云端数据库)。
  - **功能职责**: 提供云端数据库存储，接收前端通过 API 发送的增、删、改、查指令，确保数据跨设备永久留存。
- **部署与版本控制 (Deployment & CI/CD):**
  - **版本控制**: Git (用于记录代码演进过程，随时回滚试错)。
  - **线上托管**: GitHub Pages (将前端静态代码免费部署为公网可访问的真实网站)。



### 业务需求 (Business Requirements)

#### 1. 地图的视觉呈现 (Visual Presentation)

本项目的核心视觉体验围绕“平滑下钻”、“行政区划填色”以及“按同行者分类的色彩映射”展开。

- **核心渲染与多级下钻 (Drill-down Experience):**
  - **一级视图（世界地图）**：默认展示带有光影质感的 3D 地球。支持鼠标拖拽旋转和缩放。
  - **二级视图（国家地图）**：点击某个国家（如中国、日本），视角平滑放大并切换至该国家的 2D 行政图，展示该国省/州级边界。
  - **三级视图（省份地图 - 仅限中国）**：在中国 2D 地图上，点击特定省份，平滑切换至该省的市级行政图。
- **性能与加载策略 (Performance & Loading):**
  - **差别化加载机制**：为保证极致体验，网页初次加载时将直接全量打包“世界地图基础框架”、“中国省级 GeoJSON 数据”以及“中国市级 GeoJSON 数据”。
  - **懒加载 (Lazy Load)**：除中国外的其他国家省级边界数据，仅在用户点击进入该特定国家时，临时向网络请求并渲染。
- **足迹渲染与多色彩映射 (Color Fill & Mapping):**
  - **基础渲染模式**：采用区域填色（Polygon Fill），而非单一打点。仅展示行政图（无需地形瓦片）。可通过按钮一键切换“普通地图”和“足迹高亮地图”。
  - **层级填色显示控制**：在二级视图（国家地图）下，针对中国地图，用户可选择切换显示“省级填色”或“市级填色”；针对其他国家的地图，则显示“省级（州级）填色”。
  - **同行者情感色板**：足迹区域的颜色由“同行者分类”决定（例如：家人、同学、独自一人等对应不同的预设高级色系，具体由前端代码生成的色板决定），并在界面角落自动生成颜色图例（Legend）。
- **边缘情况处理 (Edge Cases - 叠加足迹):**
  - 若某一地区存在多次访问记录且同行者不同（例如：先和家人去过，后和同学去过），地图图层表面将**优先渲染最新一次访问记录所对应的颜色**。
  - **悬浮提示 (Hover Tooltip)**：当鼠标悬浮在该区域时，将弹出一个美观的半透明信息框，展示该地区的所有点亮记录及对应的同行分类。

#### 2. 足迹的数据结构 (Data Structure)

为了支撑多级地图下钻与复杂的足迹叠加逻辑，云端数据库（如 Supabase）将采用“基于行程事件 (Event-based)”的扁平化表结构。每一条记录代表一次独立的行程。

核心数据表 `footprint_logs` 包含以下字段：

- **id (唯一标识符)**: UUID 类型，系统自动生成，用于精准定位和管理数据的增删改。
- **country (国家)**: 字符串类型，必填项。用于匹配一级/二级视图的行政区划映射（如：中国、日本）。
- **province (省/州)**: 字符串类型。在中国等特定国家必填，用于匹配省级下钻与填色（如：四川省、加利福尼亚州）。
- **city (城市)**: 字符串类型。仅限支持市级数据的国家（如中国）填写，用于匹配最底层的区域填色（如：成都市）。
- **visit_date (行程年月)**: 日期类型（YYYY-MM），必填项。用于排序该地点的时间线，并决定该地区在地图上最终渲染的“最新颜色”。
- **companion_type (同行者标签)**: 枚举/字符串类型，必填项。用于决定该次行程在地图上的色彩映射（如：家人、同学、独自一人）。
- **remark (一句话备注)**: 文本类型，选填项。记录此次行程的核心记忆或心情，将在界面的悬浮提示框（Tooltip）中展示，提升情感附加值。

#### 3. 操作与交互方式 (Interaction & Operation)

**沉浸式图上录入 (Click-to-Add)**

- **触发机制**：摒弃传统的级联下拉菜单表单。用户在任意层级的 2D 行政图上直接点击目标区域（例如在四川省地图上点击成都市）。
- **智能预填**：点击后页面居中弹出录入对话框，系统自动抓取并锁定国家、省份、城市字段。用户仅需补充“行程年月”、“同行者标签”及“备注”。
- **实时反馈**：点击保存并同步至云端后，对话框关闭，当前图层静默刷新并对应点亮该区域。

**全局时间轴管理 (Global Timeline Management)**

- **抽屉面板**：通过界面边缘的悬浮按钮唤出“足迹簿”侧边抽屉面板。
- **全局展示**：面板内将云端拉取的所有原始行程记录，按“行程年月”倒序进行列表排列。
- **安全删除**：每条记录配备独立的删除撤销按钮，触发时强制弹出二次确认警告框，确认后抹除数据并同步刷新地图。

**定向搜索与聚合管理 (Location-Based Search & Manage)**

- **独立搜索入口**：提供专门的搜索栏组件，支持输入省级或市级行政区名称进行模糊检索。
- **地点聚合视图**：搜索并选中目标地点后，界面展开该地点的专属管理面板，集中展示该位置下的所有历史足迹、时间线及同行记录。
- **快捷增删操作**：在聚合视图下，用户不仅可以精准删除该地点的特定历史记录，还可一键点击“基于此地新建足迹”，此时呼出的录入表单将自动锁定当前搜索地点，实现脱离地图点击的快速录入。

#### 4. 权限与安全管理 (Security & Access Control)

本项目定位为高度私密的个人应用，采用“前端路由拦截 + 后端硬核级防护”的双重安全架构，确保足迹数据的绝对安全与隐私。

**隐藏式身份验证 (Hidden Authentication)**

前端网页默认不提供任何公开的注册或登录入口。系统管理员（本人）需通过访问特定的隐藏路由（如 `/admin`）或在页面特定区域进行特殊交互来唤醒登录面板，输入管理员专属口令以获取读写权限。未经授权的常规访客访问时，网页仅渲染无任何数据的空白基础地图，且所有的“添加”、“删除”及侧边栏管理控件均被强制隐藏。

**数据库行级安全策略 (Row Level Security - RLS)**

系统的最高级防线部署于 Supabase 云端数据库层面。针对所有足迹数据表强制开启 RLS 策略。数据库将主动拦截并拒绝所有未携带合法管理员鉴权令牌（Auth Token）的 API 读写请求。这意味着即使他人获取了项目的前端源码或 API 地址，也无法从物理底层窃取或篡改任何足迹数据。

**受控的只读分享 (Controlled Read-Only Sharing)**

系统支持严格受控的数据展示功能。当需要向朋友展示足迹时，管理员可生成附带高强度随机分享令牌（Token）的专属 URL（例如 `?share=加密字符串`）。当且仅当系统识别到合法分享令牌时，才会赋予该访客仅限读取的数据呈现权限，保障数据私密性的同时兼顾社交分享需求。
